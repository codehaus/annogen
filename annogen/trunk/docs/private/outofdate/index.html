JAM - The Java Abstraction Model

JAM is an object model for Java types and their associated annotations. It provides a unified superset of the features offered by similar APIs (such as Reflection or X/Javadoc). (read more...)

Downloads

JAM is currently distributed in binary form as part of the Annogen framework.

About

JAM has evolved as part my work on XML-Beans at Apache, which I do on behalf of my employer, BEA Systems. In XML-Beans, I need to be able model Java types in order to generate XML schemas from them. I found all of the existing Java object models (x/javadoc, reflection, qdox) to be lacking in some respect. They each have some features I wanted, but none of them brought all of those features together.

JAM is now proving to be a generally useful technology in its own right. Most notably, it is already being used in Cedric Beust's EJBgen and SGen. Accordingly, I am now exploring the possibility of spinning it up as a distinct open source project, but it doesn't yet have an official open source community. Until it does, my website here is serving as a temporary home.
